# Development

## Common Commands

```bash
npm ci
npm run build
npm run install:emery
```

Use `--vnc` for emulator commands in headless sessions.

## Settings Flow

The companion settings page is generated by `src/pkjs/config_page.js`.
It stores provider, base URL, authentication, cache, and a JSON `CameraList`
in app local storage. Older `{name, stream}` entries and fixed-slot settings
are migrated on startup. URL generation and request headers live in
`src/pkjs/snapshot_source.js`.

Unauthenticated camera tests use browser image loading so previews work without
depending on cross-origin XHR access. Authenticated tests are deferred until
the user saves and requests a frame from the watch because an HTML image cannot
attach Basic, Bearer, or `X-API-Key` headers.

## Message Flow

1. The watch sends `RequestFrame`, `RequestSeq`, and `CameraIndex`.
2. PebbleKitJS generates the provider snapshot URL and authentication options.
3. Frigate account authentication logs in through `/api/login`, retains the
   session cookie, and retries once after a snapshot returns HTTP 401.
4. JS decodes the JPEG, fits it into the Time 2 viewport, and quantizes pixels to Pebble `GColor8`.
5. JS sends frame metadata followed by byte chunks.
6. The watch ignores stale sequence numbers, copies a complete frame into a runtime bitmap, and redraws.

## Release Checklist

```bash
npm run clean
npm ci
npm test
npm run build
npm run --silent release:notes -- <version>
```

Then install `build/PinHole.pbw` on a device or emulator and verify:

- Settings page opens and saves.
- Camera list appears on the watch.
- SELECT refreshes the active camera.
- UP/DOWN switch cameras and fetch a frame.
- Existing frame remains visible while refreshing.

Push a `v<package version>` tag only after these checks pass and the matching `CHANGELOG.md` entry has been reviewed. The tag-triggered GitHub Actions workflow repeats the tests and build, then uses that exact changelog entry for the GitHub Release and attaches the PBW.
